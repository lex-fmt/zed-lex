Initial Zed extension for the Lex document format.

- Tree-sitter highlighting via `lex-fmt/tree-sitter-lex` (pinned at the
  v0.9.1 commit). Highlights, injections, textobjects, outline, brackets,
  and indents queries — written in Zed's `@title` / `@emphasis` /
  `@text.literal` capture vocabulary, distinct from the `@markup.*`
  scopes upstream tree-sitter-lex queries use for nvim/Helix.
- Language injection for verbatim blocks closed with a recognised
  language annotation (`:: python ::`, `:: json ::`, …). 13 languages
  × 6 content-types = 78 patterns, generated by
  `scripts/gen-injections.py`.
- LSP integration: the extension downloads `lexd-lsp` from
  `lex-fmt/lex` releases on first use and caches it in the extension's
  working directory. The pinned version lives in `shared/lex-deps.json`.
  `lsp.lex-lsp.binary.path` in Zed's settings.json overrides the
  bundled binary for local LSP development.
- Outline / symbol picker over document title, sessions, and
  definitions; diagnostics, formatting, and code actions all flow
  through the LSP.
- bats-driven test suite (29 tests) shared with the pre-commit hook
  and CI. `scripts/test-all` is the single source of truth.
- `scripts/build` produces the publishable WASM and (with `--package`)
  a `zed-lex-<version>.tar.gz` for sideload / offline install.
